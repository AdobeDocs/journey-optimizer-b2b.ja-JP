# AIを活用した意思決定

AI意思決定は、Adobe Journey Optimizer B2B Primeの背後にあるインテリジェンスレイヤーです。 AIによる意思決定では、あらゆるブランチを静的なルールとして事前に構築するのではなく、ジャーニーメンバーシップ、エンゲージメント履歴、インテントスコア、リアルタイムの行動など、プロファイルのコンテキストを継続的に評価し、次善のアクションやパスを決定します。

この継続的な評価により、オーケストレーションは静的なルールから成果に基づく自動化に移行します。つまり、事前にあらゆる条件を定義するのではなく、望む結果を記述し、システムが一人ひとりに対してそこにたどり着く方法を決定します。

## 機能 {#capabilities}

AIを活用した意思決定は、次の機能で構成されています。

| 機能 | 決定するもの |
|---|---|
| [ジャーニートラフィック制御](../marketing/journey-traffic-control.md) | 現在どのジャーニーに属すべきかを判断できます。 個人が複数のジャーニーに適格である場合、ジャーニートラフィック制御は、もはや適合しないパスに顧客を残すのではなく、プロファイルや行動が変更された瞬間に顧客を再ルーティングします。 |
| [次善のパス &#x200B;](../marketing/next-best-path.md) | ジャーニー内の個人に最適なパス。 フィルター条件をハードコーディングするのではなく、自然言語で意図を記述し、各訪問者を最適なパスにタイミングよく誘導します。 |
| [送信時間の最適化](../marketing/email-send-time-optimization.md) | 全員に決まったスケジュールではなく、過去のエンゲージメントにもとづいて、各受信者に最適な送信ウィンドウ。 |
| コンテキストコンテンツ | コンテンツ概要から自動的に生成された、パーソナライズされた電子メールのバリエーション。ジャーニー内の単一のパーソナライズされたメールアセットとして使用できます。 _近日公開予定。_ |
| [Brand Concierge](https://experienceleague.adobe.com/ja/docs/brand-concierge/content/home){target="_blank"} | チャットやライブアシストなど、リアルタイムの会話型ルーティングと対応。 Brand Conciergeには、追加の製品使用権限が必要です。 |

これらの機能のいくつかは、同じジャーニー内のさまざまな問題を解決します。 ジャーニートラフィック制御は、複数のジャーニーが同時に同じ人物にエンゲージしようとしているときに、どのジャーニーが優先されるかを決定します。 その他の機能は、ある人がすでにジャーニーにいる後に何が起こるかを決めます。

## パスと次善のパスの分割 {#split-paths-next-best-path}

[&#x200B; パスの分割](../marketing/split-merge-paths-nodes.md)を使用すると、明示的なフィルター条件でジャーニーブランチを定義できます。スコアが閾値を超える場合は、あるパスにユーザーを送信し、それ以外の場合は別のパスに送信します。 このルールベースのロジックは正確で完全に監査可能ですが、新しい条件ごとに新しいブランチが必要です。

[次善のパス &#x200B;](../marketing/next-best-path.md) ノードは、同じ問題に対してAIによる意思決定を適用します。 このモデルでは、しきい値を固定するのではなく、エンゲージメント、ペルソナ、製品の関心度、funnelのステージを同時に評価し、各個人に対する最適な結果を予測して、最適なパスを自動的に選択します。

## データ入力 {#data-inputs}

AIによる意思決定では、次のカテゴリーのデータを利用します。

| カテゴリ | 例 |
|---|---|
| 企業特性 | 役職、業種、企業規模 |
| 心理的特性 | リードスコア、緊急性、優先度 |
| エンゲージメント | スコア、レベル、トレンド、最後のアクティビティ、チャネル |
| 目的 | 製品またはキーワードの意図を、高、中、低のバケットにグループ化 |
| ペルソナ | 案件、役職、ペルソナにおける役割 |

## 評価頻度 {#evaluation-cadence}

AIによる意思決定では、ふたつの異なる評価スケジュールを使用します。 人物の基礎プロファイルが毎晩バッチで再計算されます。 決定自体は、最新の夜間プロファイルが示すものを使用して、あらゆるインタラクションでリアルタイムで適用されます。 そのため、AIによる意思決定の連続的な部分では、プロファイルの更新率ではなく、決定の瞬間を記述します。

## ガードレールとルール {#guardrails-rules}

ルールは、誰かが明示的に書き留めた条件のみをカバーします。 ハイブリッドペルソナや誰も予想しなかったシグナルの組み合わせなど、現実がそのツリーの外側に落ちると、誰かが新しいブランチを追加するまでルールは何もしません。 AIによる意思決定では、あらゆる顧客を継続的にスコアリングし、信頼にもとづく最適なアクションを出力します。そのため、誰も明示的にプログラムしていない組み合わせを処理し、より多くの結果を見出すにつれて改善します。しかし、ルールではそのような結果は得られません。

ルールは説明可能で、監査がすぐに可能です。一方、AIによる意思決定は、決定論的ではなく信頼性スコアが付きます。 そのため、次のような場合にはルールが最適なツールです。

* オプトアウトされた連絡先の除外など、厳格なコンプライアンスの境界を超えてはなりません。
* モデルが学習するのに十分な量や履歴がありません。
* あらゆる決定は、必要に応じて完全に説明できるものでなければなりません。

成熟度の高い設定の多くは、ルールをガードレールとして、AIを利用して判断するという具合に、併用されています。

## メリット {#benefits}

* 維持するルールブランチが少なく、手作業によるジャーニーの複雑さが少なくなります。
* 数百ものルールを構築することなく、より適切なエクスペリエンスを提供。
* 選定の効率の向上：
* 各プロファイルにおける次善のエンゲージメントをすばやく特定。

**今後のスコープ：アカウントと購買グループのコンテキスト**

現在、Journey Optimizer B2B Primeでは利用できません。 AIによる意思決定は、個人プロファイルの枠を超えて、次のことに拡張する予定です。

* アカウントコンテキスト：意思決定の入力として、企業レベルおよびアカウントレベルのシグナルを提供
* 購買グループのシグナル – 取引における役割、意思決定者または実務担当者のステータス、購買に関与するすべての関係者のエンゲージメントが集約されます。
* アカウントレベルのジャーニーオーケストレーション – アカウントまたは購買グループを単位として決定し、ジャーニーのトラフィック制御で複数の関係者を調整します。
