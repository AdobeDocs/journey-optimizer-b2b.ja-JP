# 概要からプログラムを作成する

[!DNL Adobe Journey Optimizer B2B Prime]では、[_プログラム_](../marketing/programs.md)&#x200B;は、キャンペーン内のジャーニー、ユーザーリスト、トークン、設定の最上位コンテナです。 [&#x200B; チャットインターフェイス &#x200B;](./chat-interface.md)から、プログラム作成スキルは、プログラム自体、サブフォルダー、トークン、人物リスト、人物ジャーニーなど、概要からエンドツーエンドで、単一のガイド付き会話で構造全体を構築します。

* **スキル** - `program-creation`
* **招待状** – 概要をアップロードして&#x200B;_この概要からプログラムを作成_ / _概要からプログラムを作成_&#x200B;するか、キャンペーンを直接記述します
* **読み取り/**&#x200B;に書き込む – [!DNL Journey Optimizer B2B Prime]。テナントからプログラムタイプ設定も読み取ります

## ブリーフのアップロード

チャット入力の「_添付_」アイコンをクリックしてファイルをアップロードし、必要な内容を記述します。 概要のテキストが抽出され、アシスタントにコンテキストとして渡されます（チャットに「Campaign Brief uploaded: filename （NkB）」インジケーターが表示されます）。

### サポートされているファイルタイプ

| 書式設定 | 動作 |
|---|---|
| `.txt`, `.md` | ブラウザーで直接読み込む |
| `.pdf`, `.docx`, `.xlsx`, `.json`, `.csv` | バックエンドのテキスト抽出サービスに送信 |

*brief*&#x200B;を含む名前のファイルは、自動的にキャンペーン概要としてタグ付けされます。それ以外の場合は、アシスタントがコンテキストから推測します。

### サイズ制限

概要テキストは、アシスタントに到達する前に、約30,000文字（約7.5K トークン）に切り捨てられます。 その時点で、非常に長いブリーフが切り取られます（`(truncated to 29KB)`個のメモで示されます）。 キャンペーンに関する重要な詳細情報を事前に伝える。

## 概要の推奨

フロー取り込みでは、次の入力についてブリーフを読み取ります。可能な限り多くのデータを含めます。

* プログラム名と簡単な説明/目的
* プログラムタイプのシグナル（例：トレードショー、イベント、ウェビナー、ロードショー、会議、ナーチャリング）
* ピープルリストのオーディエンス基準
* ジャーニーデザイン：数、入力基準、顧客接点/タイミング、公開日
* トークン（イベント日、会場、件名などの再利用可能な値）

AI Assistantは、ビルド前に欠落しているものをリクエストします。

## フェーズを構築

スキルは、次の3つのフェーズで実行されます。**取り込み→承認→ビルド**。 承認するまで何も作成されません。

### 受注

AI アシスタントは、ブリーフから次の要素を抽出し、欠けているデータを入力します。

* プログラム名
* 親フォルダー（指定しない場合はデフォルトでワークスペースのルート）
* 説明
* プログラムの種類（可能な限り簡潔な表現から推測）
* 作成するトークン（名前、タイプ、値）
* ユーザーリストのオーディエンス基準
* ジャーニー：数、エントリ基準、タッチポイント、公開日

### 承認

次に進む前に、アシスタントが確認の概要を表示します。

`I'll create {program} in folder {id}, with {N} token(s), {N} dynamic people list(s), and {N} journey(s). Shall I proceed?`

明確なゴーサインで応答するまで、作成ツールは呼び出されません（_proceed_、_go ahead_、_build it_、_approved_、_yes_&#x200B;など）。

### 作成する

ビルドステップは固定の順序で実行されます。各ステップは、前のステップで生成されたIDによって異なります。

| 手順 | 説明 | ツール | 出力 |
|---|---|---|---|
| **1. フォルダー** | 親フォルダーを名前で解決するか、Workspace ルートを使用します。新しいサブフォルダーを作成できます | `getRootTree`, `browseFolders`, `createFolder` | フォルダー名+ ID |
| **1.5. プログラムの種類** | テナントプログラムの種類を検索し、概要の一致を選択します | `browseProgramTypes` | 型名+ `programTypeId` |
| **2. プログラム** | 選択したタイプにバインドされた解決済みフォルダーの下にプログラムを作成します | `createProgram` | プログラム名、ID、タイプ |
| **3. トークン** | プログラムの各`my.*` トークンを作成します | `createProgramToken` | トークン名 |
| **4. ユーザーリスト** | オーディエンス基準から属性ベースのルールを生成し、動的な人物リストを作成します | `generate_ruleset`, `createSmartListWithRules` | リスト名+ `smartListId` |
| **5. ジャーニー** | 手順4のリストに接続された人物オーディエンスエントリノードを使用して、各人物ジャーニーを作成します | `create_journey` | ジャーニー名、ID |
| **6.**&#x200B;を公開 | 公開日が設定され、確認された場合、ジャーニーを公開またはスケジュールします | `publish_journey_with_dates` | ライブ/スケジュールされたジャーニー |
| **7. QA** | オプションの検証プレビューと完全なQA チェック | `qa_program_preview`, `qa_program` | 成功/失敗/警告レポート |

**_Step依存関係:_**

* トークン、リスト、ジャーニーを添付する前に、プログラム ID （ステップ 2）が必要です。
* ステップ 4の`smartListId`は、ステップ 5の`create_journey`に渡されます。省略すると、ジャーニーのオーディエンスエントリノードは空のままになります。
* ジャーニープロンプトは常に&#x200B;_入力されたエントリノードを確保するために、エントリポイント_&#x200B;として人物オーディエンスノードで開始します。

## 出力アセット

### プログラム

最上位のコンテナ。 詳細ビューには、次のタブが表示されます。

| タブ | コンテンツ |
|---|---|
| **概要** | 名前、説明、プログラムの種類、ステータス、フォルダーの場所、タイムスタンプ |
| **属性** | タイプ定義カスタムフィールド（プログラムタイプが有効な場合のみ） |
| **トークン** | このプログラムのスコープを持つ`my.*` トークン |
| **ジャーニー** | プログラム内に含まれるジャーニー |

### トークン

プログラムのスコープに指定された`my.*`個の値（例：`my.eventDate`）を再使用できます。 それぞれに型（`text`、`date`、`rich text`、`score`、または`number`）と値があります。 プログラムのスコープ内で電子メールやコンテンツ内で解決します。

### ユーザーリスト

プログラムの下に作成された、生成された属性ベースのルールを介して、オーディエンスの基準から構築された動的な（スマートな）人物リスト。

### ジャーニー

手順4の人物リストにバインドされた人物オーディエンスエントリノードから始まる人物ジャーニーの後、概要のタッチポイントが続きます（例：_1日後にウェルカムメールを送信し、3日後にフォローアップ_）。 公開日を設定すると、ジャーニーをすぐに公開することも、スケジュールすることもできます。

## プログラムタイプの解決

プログラムは、作成時に&#x200B;**永続的に**&#x200B;をテナント定義のプログラムタイプにバインドします。これは、後で変更することはできません。 フローは、すべてのケースで`browseProgramTypes`を使用して型を明示的に解決します。

| 事例 | 動作 |
|---|---|
| **複数のタイプが使用可能** | ブリーフの文言がタイプに一致します（例：tradeshow/booth/expo = *Tradeshow*/*Event*、ウェビナー= *ウェビナー*/*イベント*、ナーチャリング/ドリップ = *ナーチャリング*、明確なシグナル = *Default*）。 一致するものが見つからない場合、アシスタントは使用可能なタイプとタスクを一覧表示します。 |
| **既定のみのテナント** | *Default*&#x200B;を使用し、管理者がカスタム [&#x200B; プログラムタイプ &#x200B;](../admin/program-types.md)を追加できることをメモします。 |
| **設定されたタイプはありません** | 停止 – 作成は失敗します。 管理者に、再試行する前にプログラムタイプをプロビジョニングするよう求めるメッセージを表示します。 |

## デフォルト

| 設定 | デフォルト |
|---|---|
| **フォルダー** | Workspace ルート（フォルダーが指定されていない場合） |
| **プログラムの種類** | *既定* （ブリーフ シグナルがなく、手動で選択しない場合） |
| **Personalization属性** | 派生ペルソナ、派生インテント、デフォルトで含まれるエンゲージメント層（オプトアウト可能） |
| **公開** | 今後の公開日には自動ではありません。手動の`ACTIVATE journey by {date}` ステップとして記載されています。今すぐ起動して確認した場合にのみ直ちに公開してください |
| **承認ゲート** | 明示的な承認の前にアセットは作成されません |

## 制限事項

| 制限事項 | 詳細 |
|---|---|
| **ブリーフの長さキャップ** | 約30,000文字。切り捨てられたブリーフは末尾のコンテンツを失い、必要なものを第一に考える |
| **不変プログラムの種類** | 作成後に変更できません。承認前に正しいタイプを確認してください |
| **必須の内部フィールド** | `parent`と`programTypeId`は常に設定されます。`parent`を省略するとアクセスが拒否されます。タイプを省略すると、自動的に&#x200B;*デフォルト*&#x200B;にフォールバックされます |
| **ジャーニーにはユーザーリストが必要です** | 手順4の`smartListId`は、手順5に必須です。フローは、これを適用します |
| **非同期リストメンバーシップ** | 静的（基準から）リストは、すぐに入力するのではなく、数分で入力されます |
| **処理エラー** | ツールのエラーは、正確なエラーとともに報告されます。アシスタントは、入力を確認して再試行するよう求められます |
| **名前の競合** | 自動解決されていません。別の名前が要求されます |
| **スコープ** | Journey Optimizer B2B Primeのみ。[!DNL Marketo Engage]人のユーザーは、プログラム作成/プログラム計画のスキルを別途使用する必要があります |


## QA チェック

ビルドが完了すると、AI アシスタントには次の機能が提供されます。

> 「起動前にQA チェックを実行して、すべてを検証しますか？」

`qa_program_preview`を実行し、その後`qa_program`を実行することを確認します。 このプロセスは、合格、失敗、警告のレポートと、推奨される次のステップを返します。
