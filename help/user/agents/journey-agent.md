# Journey Agent B2B

Journey Agent B2Bは、Adobe Journey Optimizer B2B editionのAI アシスタントであり、自然言語を通じてB2B ジャーニーを設計、実行、最適化、モニタリングするのに役立ちます。 自動化、データ主導のレコメンデーション、リアルタイムのオブザーバビリティを組み合わせることで、カスタマージャーニーの構築と管理に必要な時間と複雑さを削減します。

![Journey Agent B2B プロンプト &#x200B;](./assets/journey-agent-prompt.png)

Journey Agent B2Bでは、B2B ジャーニーのライフサイクルのさまざまな側面に焦点を当てた、一連のAI スキルを提供します。 現在利用可能なスキルは次のとおりです。

* **[ジャーニービルドスキル](#journey-build-skill)** – 自然言語プロンプトからジャーニーを作成、更新、最適化します
* **[ジャーニーの観察可能性スキル](#journey-observability-skill)** - アカウントとユーザーがアクティブなジャーニーをどのように進んでいるのかについて質問します

## ジャーニービルドスキル

ジャーニー構築スキルは、マーケティング目標、エンゲージメント戦略、KPIを、包括的なB2B カスタマージャーニーに変換します。 このレポートでは、B2B マーケターが直面している3つの重要な課題に取り組みます。

* ますます複雑なカスタマージャーニー（オーディエンス、コンテンツ、メッセージ、オムニチャネルの複雑さ）への対応
* 予算を厳格化し、効率を向上
* 最適なカスタマージャーニーをどのように構築すべきかを考える

マーケターは、このスキルを使用して、次のことを行うことができます。

* **作成** - マーケティング目標、製品、エンゲージメント戦略、KPIを、自動化と条件を使用してパーソナライズされたカスタマージャーニーに変換します
* **おすすめ** – 過去のマーケティングエンゲージメントやその他の過去データを活用して、ジャーニー作成を最適化します
* **最適化** – 予測または実際のパフォーマンスに基づいて、アクティブなジャーニーを分析、調整、最適化します
* **管理** – 重複するジャーニーとメッセージ配信の優先順位付け、管理、調整

### 基本的な使用方法

Journey Agent ビルドスキルを使用するには、プロンプトウィンドウに自然言語を使用して、作成する内容を入力します。

「B2B ジャーニーを構築し、意思決定者を、新しいパイプラインを開く可能性が最も高いエンゲージメントの高いアカウントのロードショーに招待する」。

![&#x200B; ビルドスキルのジャーニーエージェント B2B プロンプト &#x200B;](./assets/journey-agent-tasks.png)

あなたが提供することができるより多くの詳細は、より良い返信になります。 イベントや製品などを説明する既存のマーケティング資料がある場合は、それをプロンプトに貼り付けることで、担当者は目標をより的確に把握できます。

「B2B ジャーニー戦略家として行動して、多段階の顧客アカウントジャーニーを構築し、`<Solution Name>`の初期段階で意思決定者とマーケティングペルソナを育成し、エンゲージします。 目標は、匿名の訪問者を既知の連絡先に変換し、`<domain>`.com上の関連コンテンツとのエンゲージメントを深め、`<Product Name>`件のセールスアウトリーチで有望なリードを獲得することです。 既存のオーディエンスセグメントとコンテンツを活用し、電子メールや有料メディアなどのチャネルを使用します。 4～6週間の認知、検討、評価のステージをまたいで、各ステージの明確なトリガー、アクション、目標に合わせてジャーニーを構築します。 コンバージョン率、エンゲージメントスコア、デモリクエストなどのKPIを含め、構造化されたジャーニーフローとして出力を返します。」

この詳細なプロンプトには、次の内容が含まれます。

* **明確な意図**: AIに対して何をすべきか？ タスクや結果を具体的に説明する：
* **豊富なコンテキスト**：関連する背景または制約を指定します。 可能であれば、例や参照を含めましょう。
* **構造化形式**：箇条書き、番号付き手順、またはテンプレートを使用します。
* **役割の割り当て**: AIの役割を指定 – 「データアナリストとして行動…」

エージェントを使用して調整を繰り返します。最初は単純で、次に結果に基づいてプロンプトを調整します。 フィードバックループは、時間の経過とともに結果を改善します。

### エンドツーエンドのB2B ジャーニーの構築

ジャーニー構築スキルは、従来のユーザーインターフェイスではなく、会話体験を通じて、自然言語のテキストプロンプトやメタデータからエンドツーエンドのジャーニーフロー（アカウントまたは個人のジャーニー）を生成できます。

エンドツーエンドのジャーニープロンプトの例：

* クロスチャネルジャーニーを構築し、過去30日間に自社のコンテンツにエンゲージしていないアカウントを育成。
* 最も重要な購買グループの役割にパーソナライズされたコンテンツを提供することで、パイプラインを開かずに高い意欲を示しているアカウントに対して、ソリューションをクロスセルするジャーニーを作成します。
* B2B ジャーニーを構築し、意思決定者を、新しいパイプラインを開く可能性が最も高いエンゲージメントの高いアカウントのロードショーに招待します。
* web サイト上のコンテンツでエンゲージしている人々に焦点を当て、私のソリューションを提案する意思を持ってホワイトスペースアカウント向けのジャーニーを作成します。

### 多段階のジャーニー

B2B ジャーニーデザイナーとして行動し、検討段階の初期段階で、意思決定者とマーケティングペルソナに情報を提供する多段階のカスタマーアカウントジャーニーを構築できます。
匿名の訪問者を既知の連絡先に変え、関連性の高いコンテンツとのエンゲージメントを深め、セールス活動に適格なリードを絞り込むことを目指します。

* `Email`、`Paid media`、`Personalized web experiences`などのチャネルを使用して、既存のオーディエンスセグメントとコンテンツを活用します。
* 各ステージの明確なトリガー、アクション、目標を設定し、4～6週間で`awareness`、`consideration`、`evaluation`のステージをまたいでジャーニーを構築します。
* `conversion rates`、`engagement scores`、`demo requests`などのKPIを含め、出力を構造化ジャーニーフローとして返します。

## ジャーニー観察能力

Adobe Marketo Engageのジャーニーオブザーバビリティ能力を活用すれば、ジャーニーマップ、ログ、ダッシュボードを掘り下げることなく、アカウントや個人がB2B ジャーニーをどのように利用しているのか、自然言語で質問することができます。 ジャーニーの進行とデータ同期のオブザーバビリティという、2つの主要な領域について説明します。

Journey Optimizer B2B editionでは、次の2つの場所からアクセスできます。

* **ジャーニーマップの右側のパネル アシスタント** - ジャーニーマップから直接、ジャーニー固有の質問を行います。 ジャーニー名はコンテキストに自動的に挿入されます。

* **フルスクリーンアシスタントページ** – 複雑な質問やクロスアカウントの質問、複数のジャーニーの探索に適した、より大きな会話型ビュー。

### Account-level journey insight

特定のアカウントがジャーニーをどのように移動したのかを追跡し、次のような質問をすることで把握できます。

* 「ACME Industriesのアカウントに関する基本情報を教えてください。」
* 「Northstar ServicesがABM ナーチャリング戦略において、どのように成功を収めたのかを教えてください。」
* 「このジャーニーは、いつContoso LLCが開始したのですか？」

### スプリットノードのパスの推論とカウント

分岐が取得された理由と、各パスに続くユーザーまたはアカウントの数を理解します。

* 「なぜアカウント QiDemoAccount24はスプリットノード c764a9の小さな米国カリフォルニアのテクノロジーパスに行ったのですか？」
* 「スプリットノード node-459c7cの各パスの人数を教えてください。」
* 「アカウント 8のサンノゼ・パスにいる人を見せてください。」

エージェントは、分割ロジック、アカウント属性、およびアクティビティを検査して、ルーティング決定を説明し、パスごとのカウントまたは人物リストを返します。

### 個人レベルのジャーニーの結果

アカウント内またはジャーニー全体の個々の結果を分析できます。特に、購買グループの分析や意思決定者とインフルエンサーの行動の比較に役立ちます。

* 「Account123のこの分割ノードの各パスで人数を指定してください。」
* 「隠れたスキップパスのユーザーをリストアップする」
* 「このアカウントのサンディエゴのロケーションパスにいる人を表示してください。」

### ジャーニーのライフサイクルとタイミング

ジャーニーまたは特定のアカウントのキータイムスタンプを取得します。

* 「この旅行はいつ公開されましたか？」
* 「最初のアカウントがこのジャーニーに入ったのはいつですか？」
* 「アカウント QiDemoAccount50がこのジャーニーを開始したのはいつですか？」

### 外部オーディエンスとアクティベーションの可視化

LinkedInなどの外部チャネルにプッシュするジャーニーの場合、アクティベーションステータスを確認できます。

* 「bsmith@acme.comは外部オーディエンスにありますか？」
* 「ObservabilityExtAudience01を通じてLinkedInにアクティベートされたすべてのユーザーを一覧表示します。」

### データ同期とパイプラインの可観測性

オブザーバビリティ スキルでは、データ同期の正常性情報を表示することもできます。これは、アカウントまたはリードがジャーニーに含まれていない理由をトラブルシューティングするのに役立ちます。

* 外部オーディエンス書き出しのジョブ指標とステータス
* バッチセグメント化スケジュールと完了時間
* 購買グループのメンテナンスのスケジュール
* ジョブマネージャーの統計（完了/失敗）
