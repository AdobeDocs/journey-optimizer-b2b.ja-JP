# 購買グループ

B2B の販売およびマーケティングアクティビティでは、アカウントがあらゆる戦略の鍵となります。各アカウントには、人物のグループが関連付けられています。これらの人物はアカウントの従業員である場合もあれば、アカウントで作業する請負業者である場合もあります。アカウントは階層化されています。階層内の異なるレベルで異なる製品が販売される可能性があります。例えば、Adobe Experience Platform は企業レベルで最上位のアカウントに販売される場合があります。また、Adobe Photoshop は、大企業のデザイン部門など、組織内の部門や部署を表すアカウントに販売される場合があります。

![アカウントの役割に関する図](assets/account-roles-diagram.png){width="800"}

アカウント内には、_購買グループ_&#x200B;を構成するユーザーのサブセットが存在する可能性があります。これらの人々が最終的に購入の決定を行うので、マーケターからの特別な配慮が必要で、アカウントに関連付けられている他の人物とは異なる情報の提供が必要になる場合があります。購買グループは、製品ラインやオファーに応じて異なるグループのユーザーで構成される場合があります。例えば、サイバーセキュリティ製品では、通常、最高情報責任者または最高セキュリティ責任者、および法務部門の代表者に購入の承認を求める可能性があります。バグトラッキング製品では、購買グループのメンバーとして通常、エンジニアリング担当副社長と IT ディレクターを指名する可能性があります。

## 主要コンポーネント

Journey Optimizer B2B editionに購買グループを設け、セールスチームが販売を担当するソリューションのターゲットアカウントリストのメンバーを特定することで、マーケティング効果を高めることができます。 お客様とマーケティングチームが購買グループの作成を開始する前に、主要コンポーネントが定義されていることを確認します。これらのコンポーネントは、ビジネスの目標と目的を達成するために重要です。

| コンポーネント | 目的 |
| --------- | ------- |
| ソリューションに対する関心 | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>マーケティング組織として、何を販売していますか？</li><li>販売をターゲティングしている製品または製品コレクションは何ですか？</li></ul>  **_例:_** 既存顧客への新製品 X のクロスセル |
| アカウントオーディエンス | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>誰に販売しますか？</li><li>ターゲティングしているアカウントのリストは何ですか？</li></ul> **_例:_** 売上高が 100 万を超える製品 Y のアカウントで定義されたアカウントセグメント |
| 購買グループの役割テンプレート | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>ターゲティングしている役割は何ですか？</li><li>購買グループの役割に割り当てる人物を決定するのに使用されるルールセットは何ですか？</li></ul>  **_例:_** CMO の役職を持つユーザーを意思決定者の役割に割り当てる |
| 購買グループステージ | （オプション）このコンポーネントは、「購買グループは成功または失敗をどのように追跡していますか？」という質問に対する回答を提供します。 |

## メンバーの割り当て

メンバーを購買グループに割り当てる方法や、購買グループから削除する方法は 3 つあります。次のリストでは、これらの追加方法と削除方法について優先順位で説明します。最上位の方法が最も優先され、下位の方法では上書きできません。

1. **_手動アクション_** - 営業担当者が購買グループに対して手動で実行したメンバーの追加または削除アクション
2. **_ジャーニーアクション_** - [購買グループメンバーシップのジャーニーアクションノード](../journeys/action-nodes.md#add-a-people-based-action)（_購買グループに割り当て_&#x200B;または&#x200B;_購買グループから削除_）
3. **_システムジョブ_** - 購買グループの[作成](../buying-groups/buying-groups-create.md#buying-group-creation-jobs)およびメンテナンスジョブ。

購買グループ内のメンバー割当ての誤った上書きを回避するため、正確なメンバー割当てを保証するために、このリストはシステムで優先されます。 例えば、営業担当者が購買グループにメンバーを手動で追加した場合、メンテナンスジョブによってその追加内容が変更されることは望ましくありません。優先順位を使用することで、次のシナリオが適用されます。

* ユーザーが購買グループにメンバーを手動で割り当て、その後に購買グループの保守ジョブが続き、その購買グループから同じメンバーが削除される場合、保守ジョブはそのメンバーを削除せず **手動の割り当てを上書きできません**。
* ユーザーが購入グループにメンバーを手動で割り当て、その後に、同じメンバーを購入グループから削除するトリガージャーニーノードが続く場合、ノードアクション **削除しない** で、手動の割り当てを上書きできません。
* トリガーされたジャーニーアクションノードが購入グループにメンバーを追加し、その後に、同じメンバーを購入グループから削除する購入グループ保守ジョブが続く場合、保守ジョブは、そのメンバーを **削除せず**、ジャーニーアクションの割り当てを上書きできません。

## 購買グループワークフロー

1. 購買グループを作成します。

   * [ソリューションに対する関心](./solution-interests.md)と[役割テンプレート](./buying-groups-role-templates.md)を定義します。
   * [購買グループを作成](./buying-groups-create.md#create-buying-groups)し、[購買グループステージ](./buying-group-stages.md)を割り当てます。

1. 欠落している人物を完全性で特定します。

   フィルターを使用して購買グループを分析します。

   **_例:_** 意思決定者の役割が欠落しており、完全性スコアが 50 未満である

1. 購買グループの定義を完成させます。

### 購買グループの完全性スコア

完全性スコアは、購入グループに適切なメンバーが役割に割り当てられており、アカウントジャーニーで使用する準備が整っているかどうかを判断するために使用されます。 このスコアは、購買グループ内の役割の数と、1 つ以上のリードが割り当てられている役割の数に基づくパーセンテージです。

例えば、購買グループ内に 4 つの役割があり、そのうち 3 つが 1 つ以上のリードに割り当てられている場合、購買グループは 75％完成しています。

購買グループの完全性スコアは、購買グループを作成または更新するたびに再計算されます。

### 購買グループのエンゲージメントスコア {#engagement-score}

エンゲージメントスコアは、購買グループメンバーアクティビティ、重み付けアクション、重み付け役割に基づいています。 結果のスコアは、テナント/インスタンス内で正規化され、一貫性のある比較が可能になり、実用的なインサイトが得られるようになります。

初期エンゲージメントスコアの計算は、購入グループを作成するとすぐに開始され、毎日再計算されます。

エンゲージメントスコアのアクティビティと計算について詳しくは、[ エンゲージメントスコア ](./engagement-scores.md) を参照してください。
