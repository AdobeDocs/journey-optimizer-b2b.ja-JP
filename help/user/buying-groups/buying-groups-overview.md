# 購買グループ

B2B の販売およびマーケティングアクティビティでは、アカウントがあらゆる戦略の鍵となります。各アカウントには、人物のグループが関連付けられています。これらの人物はアカウントの従業員である場合もあれば、アカウントで作業する請負業者である場合もあります。アカウントは階層化されています。階層内の異なるレベルで異なる製品が販売される可能性があります。例えば、Adobe Experience Platform は企業レベルで最上位のアカウントに販売される場合があります。また、Adobe Photoshop は、大企業のデザイン部門など、組織内の部門や部署を表すアカウントに販売される場合があります。

![アカウントの役割に関する図](assets/account-roles-diagram.png){width="800"}

アカウント内には、_購買グループ_&#x200B;を構成するユーザーのサブセットが存在する可能性があります。これらの人々が最終的に購入の決定を行うので、マーケターからの特別な配慮が必要で、アカウントに関連付けられている他の人物とは異なる情報の提供が必要になる場合があります。購買グループは、製品ラインやオファーに応じて異なるグループのユーザーで構成される場合があります。例えば、サイバーセキュリティ製品では、通常、最高情報責任者または最高セキュリティ責任者、および法務部門の代表者に購入の承認を求める場合があります。 バグ追跡製品には、通常、エンジニアリング担当副社長と IT ディレクターが購買グループのメンバーとして含まれています。

![ビデオ](../../assets/do-not-localize/icon-video.svg){width="30"} [概要ビデオを視聴](#overview-video)

## 主要コンポーネント

Journey Optimizer B2B editionに購買グループを設立し、セールスチームが販売を担当するソリューションに基づいてターゲットアカウントリストのメンバーを特定することで、マーケティング効果を高めることができます。 お客様とマーケティングチームが購入グループの作成を開始する前に、主要なコンポーネントが定義されていることを確認します。 これらのコンポーネントは、ビジネスの目標と目的を達成するために重要です。

| コンポーネント | 目的 |
| --------- | ------- |
| ソリューションに対する関心 | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>マーケティング組織として、何を販売していますか？</li><li>販売をターゲティングしている製品または製品コレクションは何ですか？</li></ul>  **_Example:_** 既存顧客への新規製品 X のクロス販売 |
| アカウントオーディエンス | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>誰に販売しますか？</li><li>ターゲティングしているアカウントのリストは何ですか？</li></ul> **_Example:_** 製品 Y で収益が 1M を超える勘定科目によって定義された勘定科目セグメント |
| 購買グループの役割テンプレート | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>ターゲティングしている役割は何ですか？</li><li>購買グループの役割に割り当てる人物を決定するのに使用されるルールセットは何ですか？</li></ul>  **_Example:_** CMO タイトルを持つユーザーを意思決定者の役割に割り当てる |
| 購買グループステージ | （オプション）このコンポーネントは、「購買グループは成功または失敗をどのように追跡していますか？」という質問に対する回答を提供します。 |

## メンバーの割り当て

メンバーを購買グループに割り当てる方法、または購買グループから削除する方法は 3 つあります。 次のリストでは、これらの追加方法と削除方法を優先順に説明します。 最上位のメソッドが最も高い優先順位を持ち、下位のメソッドは上書きできません。

1. **_手動アクション_** – 販売ユーザーが購買グループに対して実行する、メンバーの手動追加または削除アクション
2. **_ジャーニー操作_** - ジャーニー [ 購買グループ メンバーシップのアクション ノード ](../journeys/action-nodes.md#add-a-people-based-action) （_購買グループに割り当て_ または _購買グループから削除_）
3. **_システムジョブ_** – 購買グループ [ 作成 ](../buying-groups/buying-groups-create.md#buying-group-creation-jobs) および保守ジョブ。

購買グループ内のメンバー割当が誤って上書きされないようにするには、正確なメンバー割当を確保するために、このリストはシステムで優先されます。 たとえば、営業ユーザーが購買グループにメンバーを手動で追加する場合、その追加を変更する保守ジョブは必要ありません。 優先順位を使用して、次のシナリオが適用されます。

* ユーザーが購買グループにメンバーを手動で割り当て、その後に購買グループの保守ジョブが同じメンバーを購買グループから削除する場合、保守ジョブはそのメンバーを削除せず **手動の割り当てを上書きできません**。
* ユーザーが購入グループにメンバーを手動で割り当て、その後に、同じメンバーを購入グループから削除するトリガージャーニーノードが続く場合、ノードアクション **削除しない** で、手動の割り当てを上書きできません。
* トリガーされたジャーニーアクションノードが購入グループにメンバーを追加し、その後に、同じメンバーを購入グループから削除する購入グループ保守ジョブが続く場合、保守ジョブは、そのメンバーを **削除せず**、ジャーニーアクションの割り当てを上書きできません。

### 購買グループの完全性スコア

完全性スコアは、購買グループが完成しているかどうかを決定することに使用されます。つまり、適切なメンバーが役割に割り当てられ、アカウントジャーニーで使用する準備が整っています。このスコアは、購買グループ内の役割の数と、1 つ以上のリードが割り当てられている役割の数に基づくパーセンテージです。

例えば、購買グループ内に 4 つの役割があり、そのうち 3 つが 1 つ以上のリードに割り当てられている場合、購買グループは 75％完成しています。

購買グループの完全性スコアは、購買グループを作成または更新するたびに再計算されます。

### 購買グループのエンゲージメントスコア

購買グループのエンゲージメントスコアは、購買グループのメンバーが実行するアクティビティに基づいて、そのメンバーのエンゲージメントを決定する数値です。

* エンゲージメントスコアの計算は、購買グループを生成するとすぐに開始されます。
* スコアの計算には、購買グループのメンバーが過去 30 日間に実行したすべてのインバウンドアクティビティが使用されます。
* 30 日間のウィンドウがあり、アクティビティの有効期限が切れると、スコアが下がる可能性があります。
* 各アクティビティの 1 日あたりのフリークエンシーキャップは 20 です。購買グループのメンバーが 1 日に同じアクティビティを 20 回以上実行した場合、アクティビティのカウントは 20 回に制限され、それ以上の回数にはなりません。
* 表示されるスコアは四捨五入されます。例えば、スコアが 75.89999 の場合、76 と表示されます。

#### 重み付け

ユーザーは、役割テンプレート内の各役割に&#x200B;_重み付け_&#x200B;を割り当てて、役割に異なる重み付けを割り当て、エンゲージメントスコアを計算できます。

![役割テンプレートの各役割に重み付けを設定](./assets/roles-templates-weighting.png){width="700" zoomable="yes"}

各重み付けレベルは値に変換され、エンゲージメントスコアの計算に使用されます。

* [!UICONTROL 全く重要でない] = 20
* [!UICONTROL 重要ではない] = 40
* [!UICONTROL 標準] = 60
* [!UICONTROL 重要] = 80
* [!UICONTROL 非常に重要] = 100

_[!UICONTROL 非常に重要]_、_[!UICONTROL 重要]_、_[!UICONTROL 標準]_&#x200B;として重み付けされた 3 つの役割を持つ役割テンプレートは、次の重み付けパーセンテージに変換されます。

| 役割 | 重み付け | システム値 | 値の計算 | 割合 |
|-------------- |--------- |------------- |------------------ |---------- |
|               |          |              |                   |           |
| 意思決定者 | 非常に重要 | 100 | 100/240 | 41.67％ |
| インフルエンサー | 重要 | 80 | 80/240 | 33.33％ |
| 実務担当者 | 標準 | 60 | 60/240 | 25％ |
|               | 合計 | 240 |                   |           |
