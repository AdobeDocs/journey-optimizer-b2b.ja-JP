# 購買グループ

B2B の販売およびマーケティングアクティビティでは、アカウントがあらゆる戦略の鍵となります。各アカウントには、人物のグループが関連付けられています。これらの人物はアカウントの従業員である場合もあれば、アカウントで作業する請負業者である場合もあります。アカウントは階層化されています。階層内の異なるレベルで異なる製品が販売される可能性があります。例えば、Adobe Experience Platform は企業レベルでトップレベルのアカウントに販売される一方、Adobe Photoshop は大企業のデザイン部門など、組織内の部門や部署を表すアカウントに販売される可能性があります。

![アカウントの役割に関する図](assets/account-roles-diagram.png){width="800"}

アカウント内には、_購買グループ_&#x200B;を構成するユーザーのサブセットが存在する可能性があります。これらは最終的に購入の決定を行うユーザーなので、マーケターからの特別な配慮が必要で、アカウントに関連付けられている他のユーザーとは異なる情報の提供が必要になる可能性があります。購買グループは、製品ラインやオファーに応じて異なるグループのユーザーで構成される場合があります。例えば、サイバーセキュリティ製品では、通常、最高情報責任者または最高セキュリティ責任者、および法務部門の代表者に購入の承認を求める可能性がありますが、バグトラッキング製品では、購買グループのメンバーとして通常、エンジニアリング担当副社長と IT ディレクターを指名する可能性があります。

## 主要コンポーネント

Journey Optimizer B2B Edition で購買グループを確立し、販売チームが販売を担当するソリューションに基づいて、ターゲットアカウントリストに欠落しているメンバーを識別することで、マーケティング効果を高めることができます。お客様とマーケティングチームが購買グループの作成を開始する前に、主要コンポーネントが定義されていることを確認します。これらのコンポーネントは、ビジネスの目標と目的を達成するために重要です。

| コンポーネント | 目的 |
| --------- | ------- |
| ソリューションに対する関心 | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>マーケティング組織として、何を販売していますか？</li><li>販売をターゲティングしている製品または製品コレクションは何ですか？</li></ul>  **_例：_**&#x200B;既存顧客への新製品 X のクロスセル |
| アカウントオーディエンス | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>誰に販売しますか？</li><li>ターゲティングしているアカウントのリストは何ですか？</li></ul> **_例：_**&#x200B;売上高が100万を超える製品 Y のアカウントで定義されたアカウントセグメント |
| 購買グループの役割テンプレート | このコンポーネントは、次の質問に対する回答を提供します。 <ul><li>ターゲティングしている役割は何ですか？</li><li>購買グループの役割に割り当てる人物を決定するのに使用されるルールセットは何ですか？</li></ul>  **_例：_** CMO の役職を持つユーザーの意思決定者の役割への割り当て |
| 購買グループステージ | （オプション）このコンポーネントは、「購買グループは成功または失敗をどのように追跡していますか？」という質問に対する回答を提供します。 |

## 購買グループワークフロー

1. 購買グループを作成します。

   オプション：
   * [ソリューションに対する関心](./solution-interests.md)と[役割テンプレート](./buying-groups-role-templates.md)を使用
   * サードパーティの読み込みを使用
   * AI／ML から生成

1. 欠落している人物を識別します。

   フィルターを使用して購買グループを分析します。

   **_例：_**&#x200B;意思決定者の役割が欠落しており、完全性スコアが 50 未満である

1. 購買グループの定義を完成させます。

### 購買グループの完全性スコア

完全性スコアは、購買グループが完成しているかどうかを決定することに使用されます。つまり、適切なメンバーが役割に割り当てられ、アカウントジャーニーで使用する準備が整っています。このスコアは、購買グループ内の役割の数と、1 つ以上のリードが割り当てられている役割の数に基づくパーセンテージです。

例えば、購買グループ内に 4 つの役割があり、そのうち 3 つが 1 つ以上のリードに割り当てられている場合、購買グループは 75％完成しています。

購買グループの完全性スコアは、購買グループを作成または更新するたびに再計算されます。

### 購買グループのエンゲージメントスコア

購買グループのエンゲージメントスコアは、購買グループのメンバーが実行するアクティビティに基づいて、そのメンバーのエンゲージメントを決定する数値です。

* エンゲージメントスコアの計算は、購買グループを生成するとすぐに開始されます。
* スコアの計算には、購買グループのメンバーが過去 30 日間に実行したすべてのインバウンドアクティビティが使用されます。
* 30 日間のウィンドウがあり、アクティビティの有効期限が切れると、スコアが下がる可能性があります。
* 各アクティビティの 1 日あたりのフリークエンシーキャップは 20 です。購買グループのメンバーが 1 日に同じアクティビティを 20 回以上実行した場合、アクティビティのカウントは 20 回に制限され、それ以上の回数にはなりません。
* 表示されるスコアは四捨五入されます。例えば、スコアが 75.89999 の場合、76 と表示されます。

#### 重み付け

ユーザーは、役割テンプレート内の各役割に&#x200B;_重み付け_&#x200B;を割り当てて、役割に異なる重み付けを割り当て、エンゲージメントスコアを計算できます。

![役割テンプレートの各役割に重み付けを設定](./assets/roles-templates-weighting.png){width="700" zoomable="yes"}

各重み付けレベルは値に変換され、エンゲージメントスコアの計算に使用されます。

* [!UICONTROL 全く重要でない] = 20
* [!UICONTROL 重要ではない] = 40
* [!UICONTROL 標準] = 60
* [!UICONTROL 重要] = 80
* [!UICONTROL 非常に重要] = 100

_[!UICONTROL 非常に重要]_、_[!UICONTROL 重要]_、_[!UICONTROL 標準]_&#x200B;として重み付けされた 3 つの役割を持つ役割テンプレートは、次の重み付けパーセンテージに変換されます。

| 役割 | 重み付け | システム値 | 値の計算 | 割合 |
|-------------- |--------- |------------- |------------------ |---------- |
|               |          |              |                   |           |
| 意思決定者 | 非常に重要 | 100 | 100/240 | 41.67％ |
| インフルエンサー | 重要 | 80 | 80/240 | 33.33％ |
| 実務担当者 | 標準 | 60 | 60/240 | 25％ |
|               | 合計 | 240 |                   |           |
