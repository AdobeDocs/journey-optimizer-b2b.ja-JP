# 単一ページアプリケーション

シングルページアプリケーション（SPA）は、ページ全体をリロードすることなくページコンテンツを動的に更新するため、web パーソナライゼーションには独自の課題があります。 Journey Optimizer B2B editionには、SPAのパーソナライゼーションを効果的に処理するための専用ツールが用意されています。

## SPAについて

SPAでは、各ナビゲーショントリガーがページ全体を読み込む従来のマルチページ web サイトとは異なり、JavaScriptを使用して、ひとつのHTMLページを管理しながら、コンテンツを動的に更新します。 このアプローチでは、web エクスペリエンスに対していくつかの考慮事項が生まれます。

* **ページの再読み込みはありません** – 従来のページ読み込みイベントをトリガーせずにコンテンツの変更が発生します。
* **仮想ビュー** - SPA内の異なる&#x200B;_ビュー_&#x200B;または&#x200B;_画面_。個別のページとして追跡する必要があります。
* **クライアントサイドのルーティング** - JavaScript ルーター（React Router、Vue Router、Angular Routerなど）は、サーバーリクエストではなくナビゲーションを処理します。
* **動的DOM** - ページ要素は、最初のページ読み込み後に作成、変更、または削除される可能性があります。

## SPA サポートの設定

SPAを効果的にパーソナライズするには、Journey Optimizer B2B editionでバーチャルビュー間を移動するタイミングを特定できるように、ビュートラッキングを設定する必要があります。

### ビュー宣言の設定

開発チームと協力して、Adobe Web SDKを使用してビュー宣言を実装します。 これらのビュー宣言を使用するには、SPAが新しいビューに移動するたびに、ビュー情報を使用して`sendEvent` コマンドを呼び出す必要があります。

**実装例：**

```javascript
// When a new view is rendered in your SPA
alloy("sendEvent", {
  "xdm": {
    "web": {
      "webPageDetails": {
        "viewName": "product-detail",
        "URL": window.location.href
      }
    }
  },
  "data": {
    "__adobe": {
      "target": {
        "viewName": "product-detail"
      }
    }
  }
});
```

### 命名規則の表示

アプリケーションの論理構造に一致する、一貫性のある記述的なビュー名を使用します。

| 表示 | 名前の例 |
| ---- | ------------ |
| ホームページ | `home` |
| 製品リスト | `product-list` |
| 製品の詳細 | `product-detail` |
| 買い物かご | `cart` |
| チェックアウト | `checkout` |
| アカウントダッシュボード | `account-dashboard` |

>[!NOTE]
>
>ビュー名では大文字と小文字が区別されます。 実装では、一貫した命名規則（ハイフン付きの小文字をお勧めします）を使用します。

## SPA向けweb エクスペリエンスの構築

SPA向けのweb エクスペリエンスを作成する際には、次のベストプラクティスを考慮してください。

### 特定のビューのターゲティング

* [web チャネル設定](../admin/configure-channels-web.md)で、SPA ルーティング構造に一致するURL マッチング ルールを設定します。

* 修正を作成する場合は、修正を適用するビューを指定します。

* 各ビューに固有の要素をターゲットにするCSS セレクターを使用します。

### 動的コンテンツの処理

SPAでは、多くの場合、最初のページレンダリング後にコンテンツが動的に読み込まれます。 次の手法を使用して、修正が正しく適用されるようにします。

#### 要素を待つ

適用する前にターゲット要素が存在するのを待つように変更を設定します。

1. 非ビジュアルエディターで、CSS セレクターを使用して修正を追加します。

1. **[!UICONTROL 要素]**&#x200B;を待機し、最大待機時間を指定します。

1. 変更は、ターゲット要素がDOMに表示されると適用されます。

#### 突然変異オブザーバーの使用

Web SDKには、動的なコンテンツを生成するために、新しい要素がページに追加されたときに検出する突然変異オブザーバが組み込まれています。 これらのオブザーバは、要素が非同期で読み込まれる場合でも、変更が適用されることを保証します。

### SPA フレームワーク

Journey Optimizer B2B editionのweb エクスペリエンスは、人気のあるSPA フレームワークと連携します。

| フレームワーク | 注意点 |
| --------- | -------------- |
| **React** | 変更は、ReactがコンポーネントをDOMにレンダリングした後に適用されます。 ターゲティングには、クラス名またはデータ属性を使用します。 |
| **Angular** | Angularの変更検出実行後に要素をターゲティングします。 レンダリングする前に`*ngIf`を含む要素をターゲットにしないでください。 |
| **Vue.js** | Vueの`nextTick`を待って、要素がDOM内にあることを確認します。 安定したターゲティングには、参照またはカスタム属性を使用します。 |
| **Next.js / Nuxt.js** | SSR/SSG ページの場合は、Web SDK ハイドレーションが完了していることを確認してから変更を行ってください。 |

## SPA パーソナライゼーションのベストプラクティス

### 安定したセレクターを使用

SPAでは、多くの場合、動的なクラス名またはIDを生成します（特にCSS-in-JS ソリューションの場合）。 代わりに、次を使用できます。

* **データ属性** - カスタムデータ属性（`data-testid`、`data-section`など）を追加します ターゲットにする要素です。
* **セマンティック HTML** - HTML構造とセマンティック要素に基づくTarget。
* **ID属性** – 可能な限り安定したID属性を使用します。

**例：**

```html
<!-- Instead of targeting dynamic class names -->
<button class="css-1a2b3c4d">Sign Up</button>

<!-- Add a stable data attribute -->
<button class="css-1a2b3c4d" data-action="signup">Sign Up</button>
```

**CSS セレクター：** `[data-action="signup"]`

### テストビュー

SPA web エクスペリエンスをテストする場合：

1. 修正が適用されるすべてのビューを移動します。

1. 次のようなユーザーフロー全体をテストします。
   * ビューへの直接ナビゲーション（ディープリンク）
   * SPA内の他のビューからのナビゲーション
   * ブラウザーの前後ナビゲーション

1. 以前に訪問したビューに戻ったときに、変更が再適用されることを確認します。

### ビューのトランジションを処理

一部のSPAでは、ビュー間のアニメーションやトランジションを使用します。 検討する：

* **タイミング** - アニメーションの移行が完了した後に変更が適用されるようにします。
* **要素の表示** – 要素は存在する可能性がありますが、遷移中は非表示にされます。
* **ちらつき** – 表示されるコンテンツの変更を避けるために、早い段階で修正を適用します。

## トラブルシューティング

SPA デザインの変更を確認する際は、次の推奨事項を使用して、一般的な問題を解決してください。

* **変更が表示されない** – 変更がSPAに表示されない場合：

   1. **ビュートラッキングを確認** - `sendEvent`呼び出しに正しいビュー名が含まれていることを確認します。

   1. **要素の存在を確認** – 変更が適用される場合、ターゲット要素がDOM内にあることを確認します。

   1. **レビューセレクター** - CSS セレクターが実際のDOM構造と一致することを確認します。

   1. **コンソールを確認** – 変更を妨げる可能性のあるJavaScript エラーを探します。

* **変更が簡単に表示され、その後消える** – この問題は通常、SPAが変更された要素を再レンダリングして置き換える場合に発生します。

   1. より特定のCSS セレクターを使用して、レンダリング間で安定した状態を維持します。

   1. 突然変異オブザーバを有効にして、要素が再作成されたときに変更を再適用できます。

   1. 開発チームと協力して、ターゲット要素に安定した属性を追加します。

* **変更の重複** – 変更が複数回表示される場合：

   1. ビュートラッキングイベントがビュー遷移ごとに1回だけ発生することを確認します。

   1. 変更がグローバルに適用されるのではなく、特定のビューに対して適用されていることを確認します。
