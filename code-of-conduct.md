# アドビの行動規範

## 誓約

オープンで快適な環境を醸成するために、コントリビューターと管理者は、このプロジェクトとAdobe Docs コミュニティへの参加を、誰にとってもハラスメントのない体験にすることを約束します。 年齢、体格、障害、民族、性自認および表現、経験レベル、国籍、外見、人種、宗教、または性自認に関係なく、また
方向：

## Standards

ポジティブな環境の創出に貢献する行動の例
の一般的な使用法には次が含まれます。

* 包括的で受け入れやすい言葉を使う
* 異なる視点および経験を尊重する
* 建設的な批判を謹んで受け入れる
* コミュニティにとって最善なことに注力する
* 他のコミュニティメンバーに対して共感を示す

参加者による許容されない行動の例を以下に示します。

* 性的な言葉や画像の使用と歓迎されない性的注意
アドバンス
* 挑発、侮辱的／軽蔑的コメント、個人的攻撃や政治的攻撃
* 公的または個人的なハラスメント
* 物理的または電子的な他人の個人情報を公開する
明示的な許可なしにアドレス
* 合理的に不適切と考えられる他の行為
プロフェッショナルな設定

## 責任

プロジェクトマネージャーは、許容可能な基準を明確にする責任があります
適切かつ公正な是正措置を講じることが期待される
容認できない行動のインスタンスへの対応。

プロジェクト管理者は、プロジェクトを削除、編集、削除する権利と責任を持ちます
コメント、コミット、コード、Wiki編集、イシュー、その他の投稿を却下する
遵守する必要があります。 また、一時的にアカウントを停止したり
不適切と思われる他の行動の貢献者を永続的に残します，
脅迫、不快、有害な行為：

## 範囲

この行動規範は、プロジェクトスペース内とパブリックスペースの両方に適用されます
個人がプロジェクトまたはコミュニティを代表する場合。 例
プロジェクトやコミュニティを表すには、プロジェクトの公式メールを使用することが含まれます
住所、公式のソーシャルメディアアカウントからの投稿、または任命された人物として行動する
オンラインイベントまたはオフラインイベントの代表者。 プロジェクトマネージャーは、プロジェクトの表現をさらに定義および明確化できます。

## 実施

虐待、嫌がらせ、またはその他の方法で容認できない行動の事例は次のようなものがあります
プロジェクトチーム（Grp-opensourceoffice@adobe.com）に連絡して報告します。 すべて
苦情は確認および調査され、その結果、
必要かつ適切であると認められる場合。 プロジェクトチームは
事件の報告者に関する秘密を保持する義務があります。
特定の実施ポリシーの詳細は別途投稿される場合があります。

行動規範を遵守しない、または実施しないプロジェクト管理者
信仰は、他の人によって決定された一時的または恒久的な影響を受ける可能性があります
重要な役割を果たします。
